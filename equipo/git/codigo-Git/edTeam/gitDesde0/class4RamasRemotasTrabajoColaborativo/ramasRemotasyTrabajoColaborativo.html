Resolviendo conflictos

regreso

Al mezclar dos ramas donde se haya editado los mismos archivos.

Lo que hace git es marcar un fallo de contenido y mandar al archivo unas lineas de texto donde se indica el cotenido de las difernetes ramas , y para solucionarlo podemos editarlo direcot en el etitor de codigo o podemos cancelar el merge 
    
        git merge --abort 
        abortar la union de ramas aun asi se hay
        solucionado el conflicto

        git add archivo_mergeado
        despues de solucionado el conflicto podemos 
        marcarlo ponerlo denueveo en el area de preparacion,
        para despues poder hacer un commit en la rama donde 
        se mergeo

Ignorando archivos

regreso

Podemos ignorar archivos que ni sea como tal parte tan importante de proyecto.
En la raiz de nuestro proyecto debemos de crear un archivo de git-ignore = .gitignore, dentro de este con solo poner el nombre del archivo o carpeta podemos ignorar los elementos, podemos usar la notacion de UNIX con los puntos y to. 
    
        requisitos.txt
        En etse caso el archivo qucon el nombre 
        requisitos.txt va a ser ignorado

Subiendo nuestros cambios

regreso

Una ves que ya se tenga la llave podemos ir al proyecto, creo que la chula es crear en blanco el proyecto en github y depues clonarlo para hacer el push
Cuando clonas un proyecto desde github se genera una rama origin, que es como el master de ahi 
    
        git push origin master
        se sube el repo a orige desde master 

        git remote -v
        vemos desde donde y a donde se envia la info
En el log podemos ver que el head apunta a master y a origin/master, es decir git ya sabe en este momento que este conetenido pertenece a la rama local y esta está conectada con origin del remoto

Al hacer un nuevo commit tambien se debe de empujar desde master hasta origin, no se si se pude cambiar el head a origin master.

En el caso de que en la nueve haya un cambio podemos hacer 
    
        git fetch origin
        trae los cambios de origin pero en la rama de 
        combinacion de origin/master

        git pull origin master
        trae lo de origin master
