Opciones de git add y git commit

regreso

LOS tre estados siempre juegan una papel importante, recoredando que son estado de trabajo, area de preparacion y repositorio, el primero es donde pasamos modificando e archivo que hemos selccionado , despues podemos pasarlo a el area de preparacion, que cuando lo decimadamos podemos pasarlo a el area del repositorio. 

    
        git add 
        Para poder añadir los cambios, en el area de 
        preparacion.

        git restore 
        Poder deshacer los cambios ANTES de pasar a el 
        area de preparacion.

        git add .
         significa que estoy agregando todos los archivos 
        de la carpeta donde nos encontramos, es decir que si 
        existen mas carpteas no se agregaran los archivos que 
        se encuentren en ella 

        git commit -m ""
        podemos realizar el mensaje de una para que no se
        abra el editor de texto

Modificando la estructura del proyecto

regreso

En caso de que se modifique la estructura del proyecto podemos ver que la checar el status con git status, que es como si se hubiera borrado de la carptea actual y podemos agragarlo a la area de preparaciona asi sean las nuevas capetas con git add . desde al imagen de la raiz.

Archivos modificados y nuevos

regreso

Cuando tenemos un grupo de modificados podemos añadirlos directamente al repositorio con 
    
        git commit -am "Se modificarion los archivos respecto al curso de git io que se !!!!!"
        poder pasar el add y commit de todos los archivos en el area de trabajo de una, solo los modificados, en el caso de archivos nuevos solo queda mandarlo al area de preparacion y despues al repositorio, asi como tambien con carpetas .

Deshaciendo cambios realizados

regreso

Nos arepentimos de los cambios hechos antes de ser commiteado 
    
        git checkout "archivo_reseteable.txt"
        Se deshacen los cambios realizados en un archivo trackeado , obvio antes de que este sea commiteado

        git checkout -f
        Todos los camobios realizados en todos los archivos modificados se desharan

Quitando archivos del area de preparacion

regreso

Sacando del area de preparacion, es decir revertir el git add . . 
    
        git restore --stage nobre_del_archivo_added
        quitar una archivo del area de preparacion

Ver cambios realizados en cada archivo

regreso

Entendiendo git diff para poder enteder el git log 
    
        git diff "archivo_modificado"
        ver las diferencias de la modificacion del archivo, uno es el que se encuentra en nuestro directorio de trabajo y otro que es su simil que esta en el repositorio almacenado como la ultima version del archivo

        git diff --stat aarchivo_modificado
        solo estadisticas de lo que se aha modificado
