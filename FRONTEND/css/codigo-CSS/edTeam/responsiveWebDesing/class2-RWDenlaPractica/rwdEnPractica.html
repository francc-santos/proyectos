Cuando se habla de resposive web desing, las unidaes de medida en su mayoria no pueden ser absolutas, es mejor usar relativas al crecimento del viewport y no tener que declarar una nidada por cada ajuste

Cn la ayuda del porcentaje podemos controlar el comportamiento de un elemento en diferentes tamaños dando un max-width y un min-width podemos controlar el crecimiento y con el width que sea un porcentaje del viewport. 
    
        .e1 p{
            font-size: 2em;
            background: yellow;
            margin: 0 auto;
            width: 75%;
            min-width: 320px;/* no se hace mas pequeño que esto */
            max-width: 800px;/* no crece mas de esto */
        }

Relacionadas con el tamaño de fuente,
  • em= tamaño de fuente del contexto
    • el fontsize se hereda , toma el del papa o el abuelo mas cercano que tenga un tamaño definido.
    1em =100% de la funte del contexto al cambiar el tamaño de fuente el valor de 1em se redefine , sea con un valor absolut un un relativo.
    em para cambios internos dentro de un elemento
  • rem= tamaño de fuente de la raiz= html
    • 1rem=100% de root, usar rem para layout 
    
        .e2 p{
            font-size: 2em;/* 32px */
            background: yellow;
            margin: 0 auto;
            width: 75%;
            min-width: 320px;/* no se hace mas pequeño que esto */
            max-width: 800px;/* no crece mas de esto */
        }
        .e2 span{
            font-size: 2em;/* 64px */
        }

            border-radius: .25em;
            background-color: violet;
            color: #fff;
            padding: .5em 1.5em;
            margin: 2rem;
        }

regreso

footer
  • vw=viewportWidth
    • 1vw=1% del ancho del viewport Esta no dependen del papa como porcentaje, solo se basan en el tamaño del viewport.
    el 100% de el width tambien toma en cuenta el ancho debajo de la barra
  • vh=viewportHeight
    • 1vh=1% del alto del viewport
  • vmin=minumumViewport
    • 1vmin=1% del lado menor del viewport
  • vmax=maximumViewport
    • 1vmin=1% del lado mayor del viewport 
    
        :root{
            --bar-width: 13px;
        }
        .e3 {
            padding-bottom: 100em;
            width: 75%; /* 1/2 */
        }
        .e3 footer{
            font-size: 2em;
            background: violet;
            width: 100vw; /* 1/4 */
            width: calc(100vw- var(--bar-width));
        }

Breakpoint

regreso

Son los puntos de quiebre y pueden ser arbitrarios para que un diseño se vea en cada pantalla de menera chula, boostrap y fundation usan untos de quiebre diferentes y es asi , se puede definir tu tamaño small con 300px como con 400px,

  • small: 320
  • medium: 640
  • large: 1024
  • xlarge: 1240
    
        $breakpoints:(
        small: 320px,
        medium: 640px,
        large: 1024px,
        xlarge: 1200px,
    );

    @mixin rwd($bp){
        @if map-has-key($breakpoint, $bp){
            @media screen and (min-width: map-get($breakpoints, $bp)){
                @content;
            }
        }
        @media screen and (min-width: $bp){
            @content;
        }
    }

    body{
        @include rwd(small){
            background-color: yellow;
        }
        @include rwd(500px){
            background-color: darkblue;
        }
    }
