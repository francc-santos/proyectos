¿Qué son las variables de Css?

regreso

Espacio de memoria que almacena un valor, claramente pude ser recuperado, se encuentran como custom properties

Para declararla de manera global ese debe de poner en elemento mas grande o que se encuentra mas arriba , como por ejemplo en html o mejor en la pseudoclase :root que tiene una mayor especificidad. 
    
        :root{ /* selector */
            --color-h1/* Propiedad */: red /* valor*/; /* regla */
            --box-shadow: 2px 2px 5px #000;
        }
        
        h1{
            color: var(--color-h1,purple);
            box-shadow: var(--box-shadow);
        }
Las vaiables son key-sensitive y ademas con una coma podemos establecer un fault-back que es un valor que se tomara en caso de que la variable no haya sido encontrada que ademas puede ser otra variable.

Es el hambito, el espacio o el lugar donde una variable existe y desde este puede ser recuperado su valor, fuera de este hambio la variable no existe.

En este caso el scope depende mucho del DOM, y tienen herencia desciendo el valor desde donde se declaro y se nota en el eljemplo , ya que gracias a la herencia el color de los parrafos va cambiando. 
    
        p{
            color: var(--color);
        }
        body{
            --color: purple;
        }   
        article{
            --color:red;
        }
        section{
            --color:green;
        }
Podemos llamar a las propiedades custom desde otros archivos que estn linquedos el mismo html , asi solo llamando las variables de una , inclusive se pueden llamar desde el mismo archivo Html en las etiquetas , redefinirlas , etc, etc

Variables Css vs variables Sass

regreso

Existen grasndes ventajas tanto en las variables de CSS como las de preprocesadores como SASs, recomedado usar las dos, conociendo las ventajas que nos da el uso de una como el de otras.

Pagina para probar codigo Sass

  • Las variables de SASs se puden declarar sin la ayuda de un selector y las de CSs si
    • $color: red; 
            
            body{
                background: $color;
            }
  • Las variables Sass tienen scope de bloque y las de CSS de DOM
  • Operaciones (aritmeticas vs calc())
    •         
            $En Sass
            $size: 3em;
            div{
                height: ($size * 2) + 5em
            }
        
        
            --En CSS
            :root{
                --size: 3em;
            }
            div{
                height: calc(var(--size)*2+5em)
            }
        Le decimos al navegador que lo calcule el.
  • En Sass podemos existen ciertas funciones para los valores de las propiedades
    •         
            $En Sass
            div{
                color: darken($color,20)
            }
  • Las variables Sass pueden ser interpolodas y las de CSs nop
    • Podemos costruir un selector, propiedades 
            
            $red: red;
            $green: green;
            $blue: blue;

            @each $color in $red, $green, $blue{
                .#{$color}{
                    color: $color;
                }
            }
  • Sass tiene problema al redefinir , mientras que las de Css simon que si
    •         
            --En Css, no se puede hacer en SAss
            :root{
                --color: red;
            }
            div{
                color: var(--color);
            }
            div{
                --color: green;
            }
  • Las variables Sass no existen despues de la compilacion , meientras que las de CSS simon que existen en el DOM
