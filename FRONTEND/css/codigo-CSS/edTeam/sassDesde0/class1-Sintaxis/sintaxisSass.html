¿Qué es Sass?

regreso

Sass es un preprocesador de css, y un preprocesador es un lenguaje que permite generar css a través de una sintaxis similar pero aprovechando características de la programacion.

Ejemplos de los preprocesadores mas conocidos
  • Sass
  • Stylus
  • Less
Pero el mas potente es Sass y practicamente el unico.
PostCss no es un procesador, como su nobre los dice es un procesador deeespues, y este se hace con el codigo nativo de Css.

En conclucion las ventajas de tener este preprocesador son:
  • Estructurar los estilos en modulos pero compilarlos a un unico archivo CSs resulatante
  • Programar estilos automagicamente
  • Almacenar las guias de estilo del proyecto en variables
  • Hacer que proyectos grandes sean mantenibles entre equipos de trabajo

La parte importante de Saas y por loq eu se ve son las variables, que al igualq eu cualquier lenguaje de programacion, es para que se puede usardespues dependiendo del scope de esta en un lugar mas adelante del codigo. 

    
        $size: 200px;
        body{
            // podemos realizar operaciones con las variables
            padding: $size * 2;
            $width: 20px;
            margin: $width;
        }
        p{
            margin: $width;/* Esta invocacion marca erroro porque solo podemos usar esta baraiable en el scope de la regla anterior */
        }

        /* Intterpolacion de vaiable 
            -Donde se debe usar es con la porpiedad calc()
        */

        $color : red;
        .#{$color}{
            color : $color;
        }
  • !default

  • $vaiable
    • es como se declara la variable
  • #{$variable}

Anidamiento

regreso

La posibilidad de meter selectores dentro de otros, que al final de la compilacion que dara como un selectro compuesto que denotas los hijos de la etiqueta anterior.

Buenos usos del anidamiento seria el uso de animacione que impacten en el selector padre, o tambien en el uso de media querys 
    
        /* Anidamiento */

        ul{
            margin: 0;
            display: block;
            animation: menu 1s;
        
            @media screen and (min-width: 640px){
                display: flex;
            }
            @keyframes menu {
                to{
                    color: red;
                }
            }
            /* Para llmara el papa de la anidacion se puede hacer referencia a este con ->&, se hace referenciapor la sintaxis y el espcio que deja al compilar */
            &:hover{
                background-color: lightblue;
            }
            grid-template: {
                columns: 100px 100px;
                rows: auto;
            } 
        }
La anidacion ouede ser utili en el caso de propeidades donde se tenga que escribir mucho, jaja

Tipo de datos

regreso

Como sass se acerca a un lenguaje de programaion , o un super set de css, por esto tiene tipo de datos, y es importante entenderlos porque cuando se declaran funciones, mixins, operaciones, necesitaremso de validaciones
  • Strings
    • Puden ser entrecomillados o sin entrecomillar
  • Numeros
    • Pueden tener unidades; px em rem % vw vh deg, etc
  • Colores
    • raro, pero con la ayuda de funciones podemos calcular nuevos colores a partir de una base
  • Booleans
    • true or false ya tu sabe, en condicionales , no se ahi looç dejo
  • List
    • valores separados por espacio o por comas, podemos usarlas pra definir la propiedad de una caja, un background seteando los valores del shordhand, etc.
    haciendo posible que se pueden recorrer con un ciclo
  • Maps
    • podemos generar un objeto de configuracion y con una funcion map-get($map,key) podemos obtener el valor de una llave. 
    
        /* Observando los tipos de datos */
        $string: quote(hola);
        $number: 20em;
        $color: darken(red, 30);
        $boolean: true;
        $listas: 1px solid #000;
        $map: (
            color: red,
            font: arial,
            size: 20px
        );
        body{
            background: type-of($value: $string);
            content: $string;
            font-size: $number/10;
            font: map-get($map,font);
        }

@import

regreso

Uso de otros archivo para tener una mejor estructura del codigo css

Podemos indicar al compilador que no genere archivo nuevos de las importaciones con _archivo.scss
La declaracion de las variables deben de estar antes de la importacion. y por esto se acostumbra una estructura donde haya varios directorios donde ponesmo comoponentes, botones , etc y ademas se acostumbra tener un archivo config con las variables necesarias para el proyecto. 
    
        /* Uso de la importacion */
        @import "menu";

        /* _menu.scss asi indicamos qeu este archivo no se debe de compilar como un arhivo sino que se quede en donde se importa, en variables se deben de declar las variables antes de la importacion */
