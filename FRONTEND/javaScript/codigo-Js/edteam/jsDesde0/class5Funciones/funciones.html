Sintaxis Basica

retachar

Bloques de codigo que son reutilizabeles y pueden ser ejecutados las veces que nosotros queremos, y en cualquier lugar de nuestro codigo. 
    
        function saludar() {
            console.log("Hola mi niño como has estado");
        }
        
        saludar()
Esto sintaxis es solo la declaracion de la funcion y la llamada de la funcion en la ultima linea.
Mas al declarar una funcion lo que hacemos es como enseñarle al programa , el cmportamiento que va a adquirir en caso de que se le diga la accion que se debe de ejecutar.

Funciones de primera clase

retachar

Ciudanos de primera class, en lenguajes de programacion estos estan conformados por los tipos de datos primitivos.o aquelos que puden ser almacenados dentro de una variable y una funciona puede ser almacenada en una varaible.
Cundo una funcion devuleve un valor de tipo primitivo en teori esta psa a ser un dato, el que se pude guardar en una variable pero esto junto a su paso de valores.
Si solo se asigna una funcion a una variable, esta variable pasa a ser una funcion la cualtambien puede ser ejecutada y esto no es considerado un ciudadano de primera clase. 
        
            function obtenerNombreCompleto(nombre, apellidos) {
                return `${nombre} ${apellidos}`
            }
            
            let nombreCompleto = obtenerNombreCompleto("yis", "santos")
            let nonbreCompleto = obtenerNombreCompleto
            
            console.log(nombreCompleto);
            console.warn(typeof nombreCompleto); /* string = ciudadano de primera class */
            
            console.log(nonbreCompleto);
            console.warn(typeof nonbreCompleto); /* function */

Funciones anonimas

retachar

Son aquellas que no tienen nombre, solo escribimos lo parametros que vamos a necesitar en la funcion y la accin que se va a ejecutar con los parametros resividos.
  • declaramos function
  • sin dar nombre a nuestra funcion decims que va a esperar los parametros
  • ejecutamos la accion
Como veiamos en el apartado anterior sabemos que cualquier funcion puede ser almacenada en una varaiable, o para ser mas especificos la declaracion de la funcion, es por esto que este tipo de funciones se encuentran declaradas pero no nombrada y pueden ser asignadas a variables. 
        
            let sumar = function(a,b,c){
                return `suma de ${a}, ${b}, ${c}, = ${a+b+c}`
            }
            
            console.warn(typeof sumar);
            console.log(sumar(1,2,1));
Tambien no serven para ejecutar ua funcion, osa podemos hacerlas anonimas autoejecutables, esto deja el scope fuera , es como este es my mundo y ahi haces lo que quieras interprete de js 
        
            (function(a,b){
                console.log(`Resultado autoejecutado ${a+b}`); 
            }(2,4))
Los primeros parentesis indican que la funcion se va a ejecutar y los segundos es donde se le pasan los valores de los parametros que esperan.
