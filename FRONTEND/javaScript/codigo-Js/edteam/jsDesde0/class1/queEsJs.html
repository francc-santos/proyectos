Origen de js

regreso

Cundo se empezaba a usar los navegadores de la web, corrian aplicaciones web, con lengujes que corrian de lado del servidor como java y c++.

Como el servidor era el que respondia, aveces era lento para la experiencia de usuario. ps tenia que validar los datos desde el servidor que ademas tenia que entregar la pagina.

Brendan Eich en el 95, queria que la segunda version de netscape tendria que tener un motor para poder correr un lenguaje programacion que se pudiera correr desde el navegador y asi no depender tanto del servidor.COn el primer nombre de mocha, luego livescript, para despues javaScript.

MOtor v8

regreso

JS es un lenguaje just in time, es decir que se ejecuta mientras va corriendoo tambien como que es compilado en tiempo de ejecucion, son terminos usados para referirse a lenguajes que no necsitan de un compilador para que se atraducido a lenguje maquina.

Para que pueda sueder, el navegador debe de implementar un motor o un interprete que pueda leer este codigo. Que estos son particularidades de los propios navegadores. Pero el mas usado es v8 que es el motor incluido en google chrome.

Cuando nosostros descargamos el navegador incluido con este vien el motor. COn esto nos podemos dar cuenta que js se puede ejecutar en cualquier ambiente donde se este incluido el motor v8 este es el caso de node que es js en el servidor.

Al sacar el motor empizan a surgir sistemas que desde la primer linea de codigo estaban corriendo javaScript con node, e incluso en la base de datos empezaban a utilizar nomenclatura de objetos de js , con lo que empieza a ser mas importante el termino fullstack o isomorficas.
