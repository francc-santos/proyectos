Comentarios

regreso

Son lineas de codigo que no son interpretadas por el navegador. Para escribir un comentario podemos hacer uso de la sigueinte sintaxis:
// = Comentario de una linea
/* */ = Comentario de bloque
/** = funciona en vscode y se acompleta solo y cada que se de un salto de linea se crea un asterisco
Consejos para las buenas practicas de los comentarios
  • NO abusar de los comentarios, no se necesita comentar todo
  • Tal vez si tu codigo necesita de muchos comentarios es porque podria ser implementad de otra forma
  • Despues de los // escribir un espacio
  • En los comentarios de bloque se puede usar un asterisco para cada linea del comentario
  • Cundo se comenta codigo en caso de que sean linea consecutivas , es mejor comentar linea por linea, siempre y cuando no se aun funcion completa

Tipos de datos

regreso

A diferencia de muchos lenguejes de programacion JavaScript no cuenta con muchos tipos de datos.
  • NUMBER

    • Tanto los de tipo entero como flotante son de tipo number

    console.log(typeof 50);
    console.log(typeof 50.6);
  • STRING

    • Una cadena de texto que se puede declarar con "",'',``, ademas de que en Js no existe el tipo de dato caracter

    console.log(typeof "Cadena de texto");/* string */
    console.log(typeof 'Cadena de texto');/* string */
    console.log(typeof `Cadena de texto`);/* string */
  • BOOLEAN

    • Son valores que solo pueden ser considerados true o false

    console.log(typeof false);/* bool */
    console.log(typeof true);/* bool */

Tipos de dato objeto

regreso

  • ARRAY
    • Otros tipos de datos que pueden ser considerados como coleccinoes de datos.O tambien puden ser vistos como datos agrupados.
    console.log(typeof ["lis","francisco","io"]);/* object */

    Resulta que es de tipo object, porque los objetos en java script son colecciones de datos/info que tiene un identificador y un valo, en un array es valor es claramente el dato y el identificador es la posicion en la que se encuentra el valor.
  • OBJETOS
    • Igualmente son tipos de datos que contienen un identificador y un valor.La diferencia con el array que tabien es de tipo objeto recordar es que a este le tenemos que decalrar el nombre del identificador para nuestro valor que queramos almacenar.
    console.log(typeof { nombre : "lis", apellido: "santos" });/* object */
  • NULL
    • Solo hace referencia a valores que no existen y de manera extraña null es tratado como tipo de dato objeto, y puede que nunca se cambie porque tal vez este todo internet contando con la validadcion de que null sea un objeto.

Declaracion y Asignacion

regreso

Existe un tipo de dato que es NO definido
console.log(typeof undefined);
Porque esto se encuentra al lado de la decalaracion y asigancion de variables?; porque estas sirven para tener espacios reservados en memeria para poder gurdar un dato durante la ejecucion de un programa.
  • DECALRACION
    • Una DECALRACION es solomente indicar al programa que quieres reservar memoria para una variable.
    let alumno
  • ASIGNACION
    • Una ASIGNACION es cuando ya se le da el valor en concreto al espacio de memoria ya reservado.
    let alumno alumno = "io"
  • REASIGNACION
    • Dar un nuevo valor a las variables, en el caso de los lengujes de programacion que son de tipado dinamico las variables pueden ser reasignadas con cualquier tipo de dato por ejemplo; una varible que antes tenia un tipo de dato string ahora puede tener un entero en este caso sun number, eso es el tipado dinamico, pero esto le cuesta mucho al navegador porque debe de gurdar mas espacio de memeoria para esperar cualquier tipo de dato. Por eso son mas rapidos los de tipado fuerte que generalmente son los lenguajes compilados que ya saben como se va a coportar cada variable que conforma el programa.
    let alumno = "io" alumno = "io2"

Asignacion por referencia

regreso

Tambien es un tema de las variables, solo que podemos asignar un valor de acuerdo a otro valor ya asignado, pero en el MOMENTo en que se haya dado el valor se convierte en variables diferentes que solo en este momento tendran el mismo valor, no es que las estemos enlazado ,sino que cuando se tenga un reasignacion normal este va tener un valor diferentes si es que no se reasigno con las misma variable cuando ya tenia otro valor. 
        
            let profesorr = "lis"
            let alumnoo = profesorr
            profesorr = "io"

            console.log(profesorr);
            console.log(alumnoo);
